Detailed Description

Abstract base class for bank accounts.

Provides common state and behavior for concrete account types (e.g., Savings, Checking). Each account has an IBAN, an owner, a balance, an interest rate, and a creation date. The creation date is initialized from GlobalClock.

Persistence:

Implements Storable for simple key-value CSV-like serialization. The marshal() encodes the owner's VAT via owner.getVat() but unmarshal() does not reconstruct the Customer reference — this must be wired externally by the caller (e.g., lookup by VAT and call a setter).

Subclassing:

Concrete subclasses must implement getAccountTypeCode() to provide a type code used in IBAN generation and serialization.
