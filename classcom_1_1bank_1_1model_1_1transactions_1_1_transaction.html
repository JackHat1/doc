Detailed Description

Abstract base class representing a financial transaction.

The Transaction class defines the fundamental structure for all financial operations performed by the banking system. Subclasses such as Deposit, Payment, and Transfer provide concrete implementations of execute() to perform specific actions on accounts.

Core responsibilities:

  • Assign a unique transactionId (UUID-based).
  • Record the timestamp of creation using GlobalClock#getDateTime().
  • Store the User who initiated the transaction.
  • Optionally store a StatementEntry for reporting and persistence.

◆ execute()

abstract void com.bank.model.transactions.Transaction.execute ( )
abstract

Executes the transaction logic.

Each subclass must override this method to define its own financial logic, such as modifying balances, recording statements, and handling validation.

Example implementations:

  • Deposit#execute() — adds funds to an account.
  • Payment#execute() — pays a bill and records both debit and credit sides.
  • Transfer#execute() — transfers money between two accounts.

Reimplemented in com.bank.model.transactions.Deposit, com.bank.model.transactions.Payment, com.bank.model.transactions.Transfer, and com.bank.model.transactions.Withdrawal.
