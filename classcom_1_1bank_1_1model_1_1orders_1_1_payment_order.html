Detailed Description

Standing order that pays a bill (by payment code) on its due date within a max amount limit.

PaymentOrder extends StandingOrder to automatically execute a bill payment on the bill's due date, pulling funds from a specific charge account. It tracks consecutive failures and deactivates itself after too many failed attempts.

Execution flow:

  1. Resolve bill by paymentCode via BillManager#getBillByRF(String).
  2. Resolve charge account by IBAN via AccountManager#findByIban(String).
  3. If today equals bill due date:
    • Reject if bill amount exceeds maxAmount.
    • Reject if insufficient funds in charge account.
    • Otherwise create Payment and execute via TransactionManager#execute(Transaction).
  4. On 4th failure, mark order inactive.

User user )

Executes the payment order on the specified payment day.

Executes only if paymentDay equals the bill's due date. Fails if amount exceeds maxAmount or funds are insufficient. On success, creates a Payment, executes it via the transaction manager, marks the bill as paid, and resets failedAttempts. After more than 3 failures, the order is deactivated.

Parameters
paymentDayThe day the scheduler attempts to execute the order.
billManThe BillManager used to fetch the bill by payment code.
accountManThe AccountManager used to resolve the charge account.
transManThe TransactionManager used to execute the payment transaction.
userThe user initiating the execution (auditing/ownership).
