Detailed Description

Manages issuing, paying, loading and persisting bills.

Coordinates Bill lifecycle and storage:

  • Issue bills (unique by payment code) and persist them in date-based daily files.
  • Mark bills as paid and persist them into a separate paid ledger.
  • Load bills from:
    • Global ledgers: issued.csv, paid.csv
    • Daily files: ./data/bills/YYYY-MM-DD.csv
  • Resolve issuer BusinessAccount via AccountManager and UserManager (by VAT).

Storage layout:

./data/bills/
  issued.csv              # open (unpaid) bills (isPaid stripped)
  paid.csv                # paid bills (isPaid:true)
  2025-10-07.csv          # daily snapshot(s)
  2025-10-08.csv          # ...

Issues (creates or replaces) a bill and persists it.

◆ loadBills()

void com.bank.manager.BillManager.loadBills ( )

Loads bills from global ledgers and daily files; rebuilds in-memory state.

  • Clears current lists.
  • Loads issued.csv into issuedBills (only unpaid).
  • Loads paid.csv into paidBills (forced paid=true).
  • Scans daily files and merges unseen unpaid bills into issuedBills.
  • Resolves issuer BusinessAccount for each bill via parseBill(String).

◆ payBill()

void com.bank.manager.BillManager.payBill ( Bill bill)

Marks a bill as paid and persists updates.

  • Moves the bill from issued to paid in-memory lists.
  • Updates the day file entry (still stripped isPaid).
  • Appends to global paid.csv with isPaid:true.
  • Calls saveAll() to refresh global ledgers.
