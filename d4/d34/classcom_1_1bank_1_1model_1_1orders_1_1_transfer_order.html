Detailed Description

Standing order that transfers a fixed amount between two IBANs on a specific day each month.

TransferOrder extends StandingOrder to automate periodic transfers. It executes on months where the execution date falls within the validity window and the day-of-month equals transferDay. On each failed attempt (e.g., insufficient balance), the failure counter increases; after more than 3 failures the order deactivates.

Notes:

  • transferFreq is stored/serialized but not currently enforced in execute; the implementation runs on the matching transferDay of any month within the window.
  • Consider validating negative amounts and adding null checks for accounts.

void com.bank.model.orders.TransferOrder.execute ( LocalDate trDayDate,
BillManager billMan,
AccountManager accountMan,
TransactionManager transMan,
User user )

Executes the transfer if the date matches the schedule and funds are sufficient.

Execution conditions:

  • trDayDate is within startingDate and endingDate (inclusive).
  • trDayDate.getDayOfMonth() == transferDay.
  • Source account has at least amount balance.

On success, creates a Transfer transaction and executes it via TransactionManager. On failure, increments failedAttempts. After failedAttempts > 3, the order deactivates.

Parameters
trDayDateThe attempted execution date.
billManUnused here (required by signature), kept for uniformity with other orders.

transManManager to execute the transfer transaction.
userThe user initiating/owning the order (for auditing).
