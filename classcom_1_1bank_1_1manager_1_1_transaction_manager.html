Detailed Description

Central manager for executing and tracking transactions.

The TransactionManager is responsible for executing all transaction types (Deposit, Withdrawal, Transfer, Payment) and storing them in an in-memory transaction history.

Each transaction is executed via its Transaction#execute() method, and upon successful completion, it is added to the managerâ€™s history list.

Example usage:

TransactionManager tm = new TransactionManager();
Transaction deposit = new Deposit(acc, 200.0, user, "Top-up");
tm.execute(deposit);

void com.bank.manager.TransactionManager.execute ( Transaction transaction)

Executes a transaction and records it in history.

Calls Transaction#execute() and, if successful, appends the transaction object to the internal history.

If execution fails, the error message is printed to the standard error stream.

transactionThe Transaction object to execute.
