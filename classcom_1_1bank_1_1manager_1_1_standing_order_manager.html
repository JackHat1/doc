Detailed Description

Coordinates lifecycle and execution of standing orders (payments & transfers).

Maintains active/expired/failed standing orders in memory and persists them to CSV files. Provides utilities to load orders from disk, create/delete orders, bulk-execute them for a given date, and save current state.

Persistence layout:

./data/orders/
  active.csv   # ενεργές εντολές (serialized μέσω StandingOrder.marshal)
  expired.csv  # ληγμένες εντολές
  failed.csv   # αποτυχημένες εντολές (λογική failure υλοποιείται στις υποκλάσεις)

Supported types:

  • PaymentOrder (type: "PaymentOrder")
  • TransferOrder (type: "TransferOrder")

Executes all orders for a given date and persists the resulting state partitions.

Για κάθε order:

  • Αν StandingOrder#isExpired() ⇒ μετακίνηση στο expired set.
  • Αλλιώς αν StandingOrder#isFailed() ⇒ μετακίνηση στο failed set.
  • Αλλιώς ⇒ StandingOrder#execute(LocalDate, BillManager, AccountManager, TransactionManager, User) και παραμένει active.

Στο τέλος, αποθηκεύει active/expired/failed στα αντίστοιχα CSV.
