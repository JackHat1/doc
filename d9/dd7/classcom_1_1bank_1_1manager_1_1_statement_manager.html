Detailed Description

Handles persistence and retrieval of account statement entries.

Each account has a separate CSV file containing its transaction statements. The manager provides methods to:

  • Save new StatementEntry lines after deposits, withdrawals, transfers, or payments.
  • Load and reconstruct full statement history for a given Account.

Storage layout:

./data/statements/
  GR100123456789012.csv   # statement entries for account GR100123456789012
  GR200987654321098.csv   # ...

Example line (CSV):

JohnDoe,GR100...,GR200...,500.0,Transfer Payment,Credit,2025-10-07 14:32:00,1500.0

List< StatementEntry > com.bank.manager.StatementManager.load ( Account acc)

Loads all statement entries for a given account.

Reads the account’s CSV file (if it exists) and converts each line into a StatementEntry instance. The list is reversed so that the most recent entries appear first.

Parameters
accThe account whose statements to load.
Returns
A list of StatementEntry objects in reverse chronological order.
