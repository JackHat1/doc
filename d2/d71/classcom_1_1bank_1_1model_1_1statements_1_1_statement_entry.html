Detailed Description

Represents a single transaction record in an account statement.

Each StatementEntry captures the essential data of a transaction — including sender and receiver IBANs, transaction type, amount, reason, timestamp, and resulting balance. It implements Storable for persistence in CSV-like textual storage format.

Usage:

  • Created and appended by TransactionManager after a successful transaction.
  • Serialized to CSV using marshal() and loaded using unmarshal(String).
  • Displayed in account statements for both debit and credit actions.

Serializes this statement entry to a CSV-compatible string.

Fields are joined by commas in this order: transactor, fromIban, toIban, amount, reason, type, timestamp, balance. Commas inside reason are replaced with spaces.
