Detailed Description

Abstract base class for automated scheduled banking orders.

A StandingOrder represents a recurring or one-time scheduled financial action (e.g., a PaymentOrder). It defines the core scheduling information, such as start/end date, title, and activation status. Subclasses implement the specific business logic inside execute(LocalDate, BillManager, AccountManager, TransactionManager, User).

Lifecycle:

  • Created with a unique orderId (UUID-based).
  • Executed periodically based on system time from GlobalClock.
  • Marked inactive after completion, failure, or expiration.

Persistence:

As it implements Storable, each subclass defines its own marshal() and unmarshal(String) to persist relevant fields.
