Detailed Description

Coordinates account loading/saving and lookups by IBAN.

Bridges the domain model (Account and subclasses) with CSV persistence via CsvStorageManager. Responsible for:

  • Loading accounts from CSV and reconstructing proper subclass instances.
  • Resolving owners/co-owners via UserManager using VAT.
  • Providing search methods (e.g., findByIban(String)).
  • Persisting all accounts back to storage with saveAll().

CSV Format Notes:

Lines are key-value pairs like: 

type:PersonalAccount,iban:GR100...,primaryOwner:EL123...,dateCreated:2025-01-01,rate:0.01,balance:5000.0,coOwner:EL999...
type:BusinessAccount,iban:GR200...,primaryOwner:EL555...,dateCreated:2025-01-01,rate:0.02,balance:1000.0,fee:5.0

◆ load()

void com.bank.manager.AccountManager.load ( )

Loads accounts from CSV into memory.

  • Reads all lines from path.
  • Parses keys (type, iban, primaryOwner, balance, rate, fee, dateCreated).
  • Instantiates the correct subclass:
    • PersonalAccount when type=PersonalAccount and owner is Individual.
    • BusinessAccount when type=BusinessAccount and owner is Company.
  • Calls Account#unmarshal(String) to fill remaining fields.
  • For PersonalAccount, resolves co-owners by VAT and adds them to the account.
Note
Lines with unknown/unsupported types are skipped. Missing or malformed values are handled best-effort; consider adding validation and error logging.
